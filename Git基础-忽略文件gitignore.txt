.gitignore称之为忽略文件，
即某些文件放到了Git的工作目录中，但这些文件又不需要提交到Git版本库中，这时就需要用到.gitignore文件了。

.gitignore文件只能作用于 Untracked Files，
也就是那些从来没有被 Git 记录过的文件（自添加以后，从未 add 及 commit 过的文件），
如果文件已经被提交到Git版本库中了，则使用.gitignore是不能忽略的。

忽略文件的原则是：
1. 忽略操作系统自动生成的文件，比如缩略图等；
2. 忽略编译生成的中间文件、可执行文件等，也就是如果一个文件是通过另一个文件自动生成的，
    那自动生成的文件就没必要放进版本库，比如Java编译产生的.class文件；
3. 忽略你自己的带有敏感信息的配置文件，比如存放口令的配置文件。

文件 .gitignore 的格式规范如下：
1. 所有空行或者以注释符号 ＃ 开头的行都会被 Git 忽略。
2. 可以使用标准的 glob 模式匹配。
3. 匹配模式最后跟反斜杠（/）说明要忽略的是目录。
4. 要忽略指定模式以外的文件或目录，可以在模式前加上惊叹号（!）取反。
5. 所谓的 glob 模式是指 shell 所使用的简化了的正则表达式。
6. 以斜杠“/”开头表示目录；
   以星号“*”通配多个字符；
   以问号“?”通配单个字符；
   以方括号“[]”包含单个字符的匹配列表；
   以叹号“!”表示不忽略(跟踪)匹配到的文件或目录；
